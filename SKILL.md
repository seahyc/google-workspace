# Google Workspace

Use this skill whenever the task is primarily about Google Workspace tools rather than local files or raw APIs.

The tool surface in this repo covers:
- Gmail
- Drive
- Docs
- Sheets
- Slides
- Calendar
- Forms

## Default approach

1. Identify the Workspace surface first.
2. Prefer the highest-level tool that already matches the task.
3. Only drop to lower-level or structured tools when the higher-level wrapper is too lossy.
4. For writes, summarize the exact target and intended mutation before acting.

## Service map

### Gmail

Use Gmail tools for:
- searching messages
- reading message or thread content
- reading attachments
- creating drafts
- managing labels
- inspecting mailbox filters

Typical flow:
1. `search_gmail_messages`
2. `get_gmail_message_content` or `get_gmail_thread_content`
3. `draft_gmail_message` or label-modification tools if needed

### Drive

Use Drive tools for:
- searching files
- reading file content
- creating files/folders
- moving, copying, exporting, or trashing files
- checking sharing state
- inspecting revisions

Typical flow:
1. `search_drive_files` or `list_drive_items`
2. `get_drive_file_content` or `get_drive_file_permissions`
3. mutate with `create_drive_file`, `update_drive_file`, `move_drive_file`, `copy_drive_file`, or export tools

### Docs

Use Docs tools for:
- reading document content
- editing text
- find/replace
- structured document edits such as lists, tables, images, headers/footers, page or section breaks
- comments and replies
- export to PDF

Typical flow:
1. `get_doc_content`
2. if structure matters, inspect with `inspect_doc_structure`
3. use `modify_doc_text`, `batch_update_doc`, or the specific insert/update tool

### Sheets

There are two layers:

- Simple table/value workflows:
  - `read_sheet_values`
  - `modify_sheet_values`
  - `format_sheet_range`

- Structured cell workflows:
  - `get_sheet_cells`
  - `update_sheet_cells`
  - `transform_sheet_cells`

Use the structured cell tools when:
- inline formatting within one cell matters
- you need to preserve or edit mixed font colors/styles
- notes or hyperlinks are part of the task
- you need CellData semantics rather than plain values

Also cover the wider Sheets surface when needed:
- spreadsheet and tab creation/deletion
- comments and replies
- borders, merges, dimensions, freezing, sorting, validation
- conditional formatting
- named ranges, filter views, protected ranges

Preferred Sheets flow for rich cells:
1. `get_sheet_cells`
2. inspect `text`, `segments`, `userEnteredFormat`, `note`, and `textFormatRuns`
3. write back with `update_sheet_cells`
4. use `transform_sheet_cells` for simple repeated text/note edits over a range

### Slides

Use Slides tools for:
- creating presentations
- reading presentation/page info
- page thumbnails
- structured batch updates
- comments and replies

Typical flow:
1. `get_presentation`
2. `get_page` or `get_page_thumbnail` if specific slide inspection is needed
3. `batch_update_presentation` for edits

### Calendar

Use Calendar tools for:
- listing calendars
- reading events
- creating, modifying, or deleting events

Typical flow:
1. `list_calendars` if needed
2. `get_events`
3. `create_event`, `modify_event`, or `delete_event`

### Forms

Use Forms tools for:
- creating forms
- reading forms
- reading responses
- publish settings

Typical flow:
1. `create_form` or `get_form`
2. `list_form_responses` or `get_form_response`
3. `set_publish_settings` if needed

## Sheets rich-cell patterns

Read:

```json
{
  "file_id": "SPREADSHEET_ID",
  "range_name": "'Sheet 1'!B2:B8",
  "facets": ["value", "formatted_value", "format", "text_runs", "notes"]
}
```

Write:

```json
[
  {
    "a1": "'Sheet 1'!B2",
    "text": "High priority follow-up",
    "base_format": {
      "textFormat": {
        "fontFamily": "Aptos",
        "fontSize": 10
      },
      "wrapStrategy": "WRAP"
    },
    "runs": [
      {
        "from": 0,
        "to": 4,
        "format": {
          "bold": true,
          "foregroundColor": {
            "red": 0.8,
            "green": 0.1,
            "blue": 0.1
          }
        }
      }
    ],
    "note": "Autogenerated by Codex"
  }
]
```

## CLI helper

Use `scripts/workspace_mcp.py` for:
- printing the service/tool catalog
- printing a per-service tool summary
- generating Sheets structured-read payloads
- generating Sheets rich-cell patch payloads
- validating Sheets patch payloads
- performing read-only Sheets structured inspection through the shared Google Workspace environment

Execution rules:
- Run the helper from the skill directory or reference it by absolute path.
- The helper resolves the Google Workspace server directory from `--server-dir`, `GOOGLE_WORKSPACE_SERVER_DIR`, or `~/.codex/config.toml`.
- When the local Python environment does not have the server dependencies, run the helper through the server environment with `uv run --directory "$GOOGLE_WORKSPACE_SERVER_DIR" python3 scripts/workspace_mcp.py ...`.

Examples:

```bash
python3 scripts/workspace_mcp.py catalog
```

```bash
python3 scripts/workspace_mcp.py service-summary --service sheets
```

```bash
python3 scripts/workspace_mcp.py sheets-build-read \
  --file-id SPREADSHEET_ID \
  --range "'Sheet 1'!B2:B8" \
  --facet value --facet formatted_value --facet format --facet text_runs --facet notes
```

```bash
python3 scripts/workspace_mcp.py sheets-build-patch \
  --a1 "'Sheet 1'!B2" \
  --text "High priority follow-up" \
  --note "Autogenerated by Codex" \
  --base-format '{"textFormat":{"fontFamily":"Aptos","fontSize":10},"wrapStrategy":"WRAP"}' \
  --run '0:4:bold=true,foreground=#cc1a1a'
```

```bash
uv run --directory "$GOOGLE_WORKSPACE_SERVER_DIR" \
  python3 scripts/workspace_mcp.py sheets-inspect-range \
  --file-id SPREADSHEET_ID \
  --gid SHEET_GID \
  --row 59
```

## Rules

- Prefer native Workspace tools over inventing ad hoc workflows.
- Prefer the structured Sheets tools when any intra-cell formatting must be preserved.
- Use simple Sheets value tools for ordinary tabular updates.
- For write operations, state the specific target and intended mutation before invoking the tool.
- Keep mutations minimal: smallest document range, cell range, or file scope that solves the task.
